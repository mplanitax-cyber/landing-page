### 1.2 01 범례 (등급·태그·채택)
큐레이션 카드를 읽기 전에 **반드시 먼저 봐야 할 룰북**.

내부 4개 영역:

1. **★ 등급 루브릭 4문항 (브리프 §4.1)**
   - Q1 GA 적합도 — 보험 GA·상담·전환 흐름 일치?

2. **Y 개수 → ★ 등급표** (5단계 색상 코드)
   - 4/4 ★★★★★ — 즉시 채택
   - 3/4 ★★★★ — 조건부
   - 2/4 ★★★ — 참고
   - 1/4 ★★ — 단편 요소만
   - 0/4 ★ — 제외 (reject_reason 필수)

3. **채택안 분류** (Yes / 조건부 / No 컬러 칩)
4. **Hard NOs** (브리프 §7) — 카피캣·실 로고·다크패턴·저작권·미출처 주장 5건

## 4. 페이지 확장 시 가이드

### 4.1 새 섹션 추가 시
Phase 5 분석에서 다른 섹션이 추가되면 (예: S17 신규):
1. 16개 카드 다음 위치에 동일 포맷의 카드 1개 추가
2. Y 좌표: `1000 + 16 * (480 + 40) = 9320` 위치 → 매트릭스 자리와 충돌
3. **해결**: 카드 17번을 매트릭스 옆 또는 하단에 별도 배치, 또는 매트릭스를 더 아래로 밀기

### 4.2 매트릭스 행 추가 시
새 브랜드가 분석 범위에 추가되면:
1. 매트릭스 프레임 높이를 `124 + N행 * 36` 로 재계산
2. 인덱스 프레임 Y 좌표 같이 밀기
3. 줄무늬 배경(zebra) 짝수/홀수 자동 처리 — `i % 2` 로직 유지

### 4.3 다국어 / 변형 페이지 만들 때
영어 버전 등을 만들고 싶으면:
- 페이지 자체는 동일 구조 유지
- 16 섹션 데이터 배열만 영어로 교체
- 색상·레이아웃·구조는 그대로 (디자인 시스템 일관성)

## 5. 주의사항 (Hard Rules)

1. **수동 편집 금지** (가능하면) — 자동 생성된 위치·크기·색상을 직접 수정하면 다음 자동 갱신 시 충돌. 텍스트 정정만 추천.
2. **흑백 강조 원칙 고정** — 컬러 도입 금지 (SKILL.md 준수). 강조는 ① 굵기 (Bold/Regular) ② 크기 (32px/16px 등) ③ 보더 패턴 (solid/dashed) 으로만
3. **★ 등급 의미 변경 금지** — 4문항 루브릭이 산업 표준. 변경 시 Phase 1~5 결과 전부 무효
4. **실 브랜드 로고 이식 금지** — 매트릭스에 텍스트만 사용, 로고 이미지 직접 가져오지 말 것 (저작권)

## 6. 자주 묻는 질문

**Q. 페이지 너무 길어 전체를 한 번에 보기 어려운데?**
A. Figma 우측 상단 줌 → "Zoom to fit" (Shift+1) 사용. 또는 Figma의 Page sections 기능으로 4개 메인 프레임을 접고 펼칠 수 있음.

**Q. 인쇄·PDF로 추출 가능?**
A. Figma의 File → Export to PDF 사용. 1440×10680 비율이라 A2~A1 사이즈 권장.

**Q. 카드 일부만 따로 빼고 싶다.**
A. 카드 프레임 우클릭 → "Copy as PNG" 또는 카드를 새 페이지로 드래그.

**Q. 색상 변경하고 싶다 (예: 브랜드 컬러 적용).**
A. **불가**. 본 페이지는 SKILL.md 흑백 와이어프레임 규칙 준수 페이지. 브랜드 컬러는 v2 컴포넌트 라이브러리 디자인 단계(Phase 9 이후)에서 적용. 현재는 흑백 유지.

**Q. 폰트는 왜 Pretendard가 아니라 Noto Sans KR?**
A. Pretendard가 본 Figma 워크스페이스에 미설치 상태. SKILL.md "Pretendard 불가 시 Noto Sans KR fallback" 규칙에 따라 자동 적용됨. Pretendard 설치 후 일괄 교체 가능.
