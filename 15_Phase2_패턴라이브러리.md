## 0. Executive Summary — 핵심 발견 6가지


### ① GA vs 직영 다이렉트 = 레이아웃 DNA가 다름

- **GA형(키움·메리츠·메타리치)**: Hero 인접 리드폼 (as_hero split 또는 right_sticky) → 상담사/상담 플로우 유도

- **직영형(삼성화재·DB·현대해상·KB)**: 대형 상품 카탈로그 카드 그리드 → AI 추천/전화 번호로 유도, 리드폼 부재 또는 간소화

- **혼합형(카카오페이·뱅크샐러드)**: 모바일-first 핀테크 톤 → 문제 정의형 Hero + 앱 설치 CTA

- **B2B형(토스인슈)**: 인슈테크 플랫폼 브랜드 사이트 → 리드 전환 아님, 참조만


**시사점**: 엠플랜잇 GA 랜딩은 **GA형 DNA**를 채택해야 함. 직영형 카탈로그 패턴은 부적합.


### ② 리드폼 position 3강 — as_hero split / right_sticky / 본문 중단 dark 이중

- **as_hero split (좌폼+우일러)**: 메리츠·메타리치·키움 (3건) — 가장 완성도 높음

- **right_sticky + 일러 배경**: 발견 못함(html.to.design 임포트 한계로 CSS sticky 평면화됨) — 원 URL에서 캐롯 확인 필요

- **이중 리드폼 (Hero split + 본문 중단 dark)**: 키움 — 전환률 2중 포획, 최우수

- **wizard 3단계**: 발견 안됨 — 브리프 우려대로 GA 리드엔 과도함 입증


### ③ 필드 수는 4개가 암묵적 표준

| 브랜드 | 필드 | CTA | 약관 |
|---|---|---|---|

| 키움에셋플래너 | 4 (이름·생년월일·성별 토글·연락처) | 원스톱 내보험 조회 | implicit |

| 메리츠화재 | 4-5 (성별 탭·이름·생년월일·연락처) | 보험료 확인하기 | inline 체크 |

| 메타리치 | 2-3 (이름·휴대폰) | 예상 확인하기 | inline 체크 |


**시사점**: 3~5 필드가 업계 표준. 6필드 이상 지양. 약관은 inline checkbox가 표준.


### ④ 로고월 생·손 분리 = 메타리치의 독보적 차별점

- 메타리치: 생명사 1줄 + 손해사 1줄 **다크블루 배경 + 흰 박스** 분리 (14+3=17개)

- 다른 GA는 단순 그리드 또는 미포함

- 시사점: **브리프 §7 variant a (생명·손해 분리)** = 메타리치 패턴 채택 권고 ★★★★★


### ⑤ 수치카드 = 직영 다이렉트의 신뢰 전략 (GA는 상대적 저약)

- 수치 pairs 많음: 삼성화재 18 / KB손보 16 / 현대해상 13 / 메타리치 7

- GA 군: 키움 1 / 메리츠 2 / 보험다모아 0 → GA는 수치 카드 적거나 없음

- 시사점: 엠플랜잇은 3~4개 정도 **핵심 지표만** 제시. 삼성화재식 18개는 오히려 산만함.


### ⑥ 가격 마스킹 = 법무 안전장치의 산업 표준

- 키움: `***,*** 원` + D/K/H 보험사 이니셜만 → 법무 리스크 최소화

- 대부분 GA는 이 패턴 사용

- 시사점: **v1 라이브러리 §11 비교카드는 price_reveal=masked로 고정**. 법무 승인 전까지 실가격 노출 금지.

## 2. ★ 굿리치 일러스트 특별 분석 — 콘텐츠↔일러스트 매핑


사용자 지시: "어떤 내용을 어떤 일러스트로 표현하는지"


### 2.1 Hero — 실사진 (일러스트 아님)

- **콘텐츠**: "고객의 삶이 더 풍요로워질 수 있도록 보다 편리한 보험 서비스를 제공합니다"

- **비주얼**: 스마트폰(보험 앱 화면 포함)을 든 사람의 **손 실사 사진** + 반투명 흰 텍스트 오버레이

- **배경**: 뉴트럴 베이지·그레이 톤, 라운드 코너 카드

- **의도 해석**: "사람(고객)의 삶"을 **실사진의 인간적 온기**로, "편리한 보험 서비스"를 **스마트폰 속 앱 UI**로 표현 — 디지털 + 휴먼 공존

- **재현성**: 실사진 라이선스 필요. 엠플랜잇은 저작권 안전 대체(스톡 이미지 또는 자체 촬영) 필요.


### 2.2 S10 통계 — 정적 SVG 꺾은선 그래프

- **콘텐츠**: "숫자로 입증하는 GA 리딩 컴퍼니" (매출 성장 곡선)

- **비주얼**: 3개 노드의 **오렌지(브랜드 컬러) 꺾은선 그래프** + 라이트 배경 + 점선 가이드

- **의도 해석**: "숫자 성장"을 **시각적 상승 곡선**으로 은유. 복잡 차트 대신 **3 point 심볼적 표현**.

- **재현성**: ✅ SVG로 5줄 이내 구현 가능 (line + 3 circles + axis dots).


### 2.3 S6 서비스 카드 5종 — 심볼릭 플랫 아이콘

각 서비스의 **핵심 기능을 시각 메타포 1개로 압축**:


| # | 서비스명 | 설명(추정) | 일러스트 메타포 | 재현 |
|---|---|---|---|---|

| 1 | 굿리치라운지 | 오프라인 고객 공간 | **지붕/집 형상** (커뮤니티 공간) | SVG, Lucide `Home` |

| 2 | 굿리치몰 | 온라인 상품 쇼핑 | **돋보기 아이콘** (탐색) | SVG, Lucide `Search` |

| 3 | 굿리치파트너 | B2B 협력사 | **체크 문서** (계약·파트너십) | SVG, Lucide `FileCheck` |

| 4 | 리치파트너 | 파트너사 | **사람 그룹 실루엣** (팀) | SVG, Lucide `Users` |

| 5 | 리치플래너 | 설계사 서비스 | **사람 1인 실루엣** (개인 전문가) | SVG, Lucide `User` |


**디자인 원칙 추출**:

- **한 서비스 = 한 상징** (다중 메타포 지양)

- **사물 > 얼굴** (추상화로 특정 인물 편향 제거)

- **일관된 선 굵기 + 2색 (그레이 + 오렌지 포인트)**

- **라운드 코너 카드 배경 + 중앙 정렬**


**엠플랜잇 적용안**: Lucide 아이콘 라이브러리 + 오렌지/블루 브랜드 컬러로 5~6 서비스 일관 매핑 → 굿리치식 심볼릭 접근은 저작권 자유도 + 재현 비용 최저.
