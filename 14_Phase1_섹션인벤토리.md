## 3. 주요 관찰·리스크

### 🟢 강점
- **Hero(S2)** 와 **Footer(S16)** 은 거의 모든 브랜드에 존재 — 수집 목표 초과 달성 예상
- **GNB(S1)** 도 대부분 브랜드에서 명확히 식별
- **FAQ(S13)**, **상품카드(S6)**, **수치카드(S5)** 는 주요 B군 브랜드(메리츠·현대·DB)에서 다수 확인

### 🟡 Phase 2 에서 보강 필요
- **리드폼(S3, 최우선 ★★★)**: 자동 분류 1~2건. 실제로는 더 많이 존재할 가능성 — 폼 필드 텍스트 단서("이름", "연락처", "약관")로 Phase 2 재탐색
- **스텝(S8)**, **상담사(S9)**, **통계(S10)**: 네이밍이 모호해 미감지 구간 존재. 일러스트 포함 Hero 안에 묶여있을 수 있음
- **페르소나(S12)**, **비교카드(S11)**: 카드 그리드로 존재 가능 — 프레임 반복 패턴(예: 600×566 Container × N개) 탐지로 2차 분류

### 🔴 블로커·한계
- **html.to.design 임포트 한계**: 원 사이트의 CSS `position:sticky`/`position:fixed`(→ S15 Sticky CTA, S1 고정 헤더) 상태가 Figma 에 평면화되어 있음. 실제 인터랙션 상태 판단은 Phase 2 에서 원 URL 재확인 필요
- **중복 2건(보맵·라이나)**: 이미 SKIP 처리. 콘텐츠가 거의 동일
- **브리프 원 대상 중 누락**: `내보험찾아줌`, 순수 `toss.im` 은 Figma 파일에 없음. Phase 2 에서 대체 여부 결정 필요
