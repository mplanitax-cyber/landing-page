## 0. Executive Summary — 한 페이지 요약

### 0.1 데이터 규모
- 분석 브랜드: **18개** (중복 2건 제외) — A군 GA 7 + B군 직영 7 + C군 핀테크 2 + EXTRA 2
- 추출된 섹션 후보 프레임: **102개**
- 시각 확인 (스크린샷): **6개 브랜드** (굿리치·메리츠·메타리치·토스인슈·삼성화재·키움)
- 메타데이터 텍스트 분석: **18개 전체** (총 813K자 XML 로컬 파싱)
- MCP 호출: 8회 (스크린샷 6 + URL 2)

### 0.2 6대 핵심 발견
1. **GA형 vs 직영 vs 핀테크 vs B2B 4분류** — 엠플랜잇은 GA형(키움·메리츠·메타리치) DNA만 채택해야 함
2. **리드폼 3강 패턴** — `as_hero split` (3건 확정), `이중 폼` (키움), `right_sticky` (캐롯 추정 — 모바일/모달 접근 패턴)
3. **필드 4개가 산업 표준** — 이름·생년월일·성별 토글·연락처 + inline checkbox 약관
4. **로고월 생·손 분리** = 메타리치 독보적 차별점 ⭐⭐⭐⭐⭐
5. **수치카드 GA 3~4개 / 직영 13~18개** — 엠플랜잇은 GA 기준 3~4개로 압축
6. **가격 마스킹 + 이니셜 브랜드** = 법무 안전장치 산업 표준 (키움 D/K/H)

### 0.3 신규 발견 (Phase 5 보강)
- **캐롯의 균일 CTA 듀얼 구조** ("보험료 확인하기" + "갱신하기") — 신규/기존 분기 패턴 ⭐⭐⭐⭐
- **캐롯의 다크패턴 의심** — "일주일간 보지 않기" 모달 약화 — `reject_reason=darkpattern` 후보
- **현대해상 할인 스토리텔링** — 모든 상품에 "○% 할인", "오프라인 대비 ○% 저렴" 강조 — 단, 현대해상은 직영 카탈로그형
- **보맵의 미니멀 전략** — 텍스트 1/3 수준, 앱+뉴스레터 중심 — GA 리드 페이지엔 부적합

### 3.2 변형 축(Variant Axis) 합의표

각 섹션의 v2 다양화를 위해 axis별로 추가 variant 추후 확장 가능:

- S1 GNB: `sticky` × `cta_count` 4 조합
- S2 Hero: `layout` × `visual` 12 조합 (현재 split×illust 1개만)
- S3 리드폼: `position` 5 × `field_count` 3 = 15 조합 (현재 2개)
- S6 상품: `columns` × `icon_style` 12 조합 (현재 3개)
- ... v2 확장 시 변형 축 한 단위씩 추가하는 식으로 라이브러리 성장

### 굿리치 일러스트 시스템에서 추출한 디자인 원칙

1. **한 서비스 = 한 상징** (다중 메타포 지양)
2. **사물 > 얼굴** (특정 인물 편향 제거)
3. **2색 일관** (그레이 + 브랜드 포인트 = 오렌지)
4. **라운드 코너 카드 + 중앙 정렬**
5. **선 굵기 일관** (Line weight 2px 추정)

**엠플랜잇 적용**: Lucide 라이브러리 + 자체 SVG로 동일 원칙 재현 가능. 저작권·재현 비용 모두 安全.
