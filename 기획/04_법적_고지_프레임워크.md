# 04. 법적 고지 프레임워크 — GA 랜딩페이지 템플릿

**작성일**: 2026-04-23
**작성자**: 시니어 기획 (Planner Agent)
**주의**: 본 문서는 **국내 표준 프레임워크 초안**으로, 실제 런칭 전 **클라이언트별 법무·준법감시 검토** 가 반드시 필요하다. 문구 오류는 보험업법·개인정보보호법 위반 과태료 대상이다.

---

## 0. 문서 목적

GA(보험대리점) 랜딩페이지 템플릿에서 **필수적으로 준수해야 하는 법령 요건**을 체계화하고, 클라이언트별로 교체 가능한 **법적 고지 JSON 필드 구조**를 정의한다.

### 적용 법령
| 법령 | 핵심 조항 | 랜딩페이지 영향 |
|---|---|---|
| 보험업법 | §87(보험대리점), §95(모집 행위), §95조의5(설명의무) | GA 역할 고지, 중개·대리 범위, 모집 관련 고지 |
| 개인정보보호법 | §15(수집·이용), §17(제3자 제공), §22(동의), §30(개인정보 처리방침), §39조의3(개인정보 국외이전) | 리드폼 동의, 처리방침 페이지 |
| 전자상거래법 | §13(신원표시), §10(전자적 표시) | Footer 사업자 정보, 통신판매업 |
| 정보통신망법 | §50(영리목적 광고성 정보) | 마케팅 동의, 수신 거부 |
| 신용정보법 | §17조의2(개인신용정보 수집·이용) | 신용정보 수집 시 별도 동의 |

## 1. 필수 고지 체계 (3층)

### 1.1 Layer 1: Footer 고정 표시
**모든 페이지** Footer에 상시 노출.

### 1.2 Layer 2: 전용 페이지
- 이용약관
- 개인정보처리방침
- (선택) 마케팅 활용 동의 약관
- (선택) 영상정보처리기기 방침 (CCTV 해당 시)

### 1.3 Layer 3: 동의 플로우
- 리드폼 제출 전 체크박스 + 모달 전문
- 필수/선택 명확 분리
- 14세 미만 처리 금지 명시

## 2. Footer 필수 표기 항목 (JSON 스키마)

### 2.1 스키마 초안
```json
{
  "legal": {
    "companyName": "(주) 엠플래닛 (예시)",
    "tradeName": "엠플래닛 보험GA",
    "ceoName": "홍길동",
    "businessNumber": "000-00-00000",
    "mailOrderNumber": "제2026-서울강남-0000호",
    "insuranceAgencyNumber": "0000000000",
    "address": "서울특별시 강남구 ...",
    "contactPhone": "02-000-0000",
    "contactEmail": "contact@example.com",
    "privacyOfficer": {
      "name": "김개보",
      "contact": "02-000-0001",
      "email": "privacy@example.com"
    },
    "complianceReviewNumber": "준법심의필 제2026-00000호",
    "complianceValidUntil": "2027-04-22",
    "hostingProvider": "AWS / (주)가비아 등",
    "complianceNotice": "본 사이트는 보험 상담 연결을 위한 정보 제공 서비스로, 보험 계약의 체결을 중개·대리하지 않습니다. 실제 가입은 보험사 및 담당 설계사를 통해 이루어집니다.",
    "privacyPolicyUrl": "/legal/privacy",
    "termsUrl": "/legal/terms",

    "partnerInsurers": 30,
    "affiliateNumber": "제2024-A-00000호"

```

### 2.2 렌더링 규칙
- 사업자등록번호, 통신판매업신고번호, 보험대리점 등록번호, 준법감시인확인필 — **한 줄에 병기** (관행)
- 대표전화·이메일 — 오른쪽 정렬 또는 별행
- 저작권 표기 © 년도 + 브랜드명
- 이용약관·개인정보처리방침 링크는 Footer 상단 별도 영역 (볼드)

### 2.3 Footer 준법 고지 예시 문구 (3 variant)

**Variant A (보수적 — 권장):**
> 본 사이트는 보험 상담 연결을 위한 정보 제공 서비스로, 보험 계약의 체결을 중개·대리하지 않습니다. 실제 가입은 보험사 및 담당 설계사를 통해 이루어지며, 보험 상품의 상세 내용은 각 보험사의 약관을 따릅니다.

**Variant B (보험대리점 등록 GA용):**
> {companyName}은(는) 「보험업법」 제87조에 따라 등록된 보험대리점(등록번호 {insuranceAgencyNumber})으로, 제휴 보험사의 보험상품을 안내·설명·상담 연결하는 서비스를 제공합니다.

**Variant C (중개·대리 행위 명시):**
> {companyName}은(는) 「보험업법」에 따른 보험대리점(등록번호 {insuranceAgencyNumber})으로, 제휴 보험사의 보험계약 체결을 중개·대리합니다. 상품의 자세한 내용과 보장 범위는 해당 보험사의 약관에 따릅니다.

> **권장**: 클라이언트의 실제 영업 범위 확인 후 A/B/C 중 선택. 기본값 A.

## 3. 리드폼 동의 플로우 (핵심 설계)

### 3.1 수집 항목 분류

| 분류 | 항목 | 근거 | 동의 필요 |
|---|---|---|---|
| 필수 | 성함, 연락처, 관심 상품, 생년월일(연령) | §15(1)(2), 계약체결 준비 | 개인정보 수집·이용 필수 동의 |
| 선택 | 성별, 상담 희망 시간, 기타 문의, 직업 | 서비스 개선 | 개인정보 수집·이용 선택 동의 |
| 제3자 제공 | 위 모두를 제휴 보험사/설계사에 전달 | §17 | 제3자 제공 별도 동의 |
| 마케팅 | SMS/이메일/카카오톡 광고 수신 | §50 | 마케팅 수신 별도 동의 (선택) |

### 3.2 동의 체크박스 UI 스펙
```
[□] 전체 동의
  [□] (필수) 개인정보 수집·이용 동의       [ 전문 보기 ]
  [□] (필수) 제3자(제휴 보험사/설계사) 제공 동의 [ 전문 보기 ]
  [□] (선택) 마케팅·광고 수신 동의         [ 전문 보기 ]
```

- "전체 동의" 체크 시 하위 전체 체크 (단, 선택은 선택)
- 필수 중 하나라도 미체크면 제출 불가 (버튼 비활성 + 안내 토스트)
- 전문 보기 버튼은 **모달** 로 원문 노출 (새창 이동 금지 — UX 단절)

### 3.3 필수 동의 전문 (표준안)

**개인정보 수집·이용 동의 (필수) 전문:**
```
{companyName}은(는) 보험 상담 서비스 제공을 위해 다음과 같이 
개인정보를 수집·이용합니다.

1. 수집 항목: 성함, 연락처, 생년월일, 관심 상품
2. 수집·이용 목적: 보험 상담 예약 및 상담 연결, 맞춤 보험 설계 제공
3. 보유·이용 기간: 수집일로부터 3년 또는 동의 철회 시까지
4. 동의 거부 권리: 귀하는 동의를 거부할 권리가 있으며, 
   동의 거부 시 상담 서비스 이용이 제한됩니다.
```

**제3자 제공 동의 (필수) 전문:**
```
{companyName}은(는) 보험 상담 연결을 위해 다음과 같이 
개인정보를 제3자에게 제공합니다.

1. 제공 대상: {companyName} 제휴 보험사 및 소속 보험설계사
   (제휴 보험사 목록: /legal/partners)
2. 제공 목적: 맞춤 보험 상담 및 상품 제안
3. 제공 항목: 성함, 연락처, 생년월일, 관심 상품
4. 보유·이용 기간: 상담 완료 후 3년
5. 동의 거부 권리: 귀하는 동의를 거부할 권리가 있으며, 
   동의 거부 시 상담 연결이 불가합니다.
```

**마케팅 수신 동의 (선택) 전문:**
```
{companyName}은(는) 다음과 같이 마케팅 정보를 제공합니다.

1. 수신 채널: SMS, 이메일, 카카오톡
2. 발송 내용: 신상품 안내, 프로모션, 이벤트, 보험 정보
3. 보유 기간: 동의 철회 시까지
4. 수신 거부: 각 채널 하단 수신거부 링크 또는 고객센터 요청
5. 동의 거부 권리: 거부 가능, 거부 시 마케팅 정보 미제공.
```

### 3.4 제출 후 처리
- DB에 **수집일시, 동의 내역(boolean), 접속 IP** 함께 기록 → 분쟁 대비
- 동의 철회 절차 안내 링크 포함 (개인정보처리방침 제N조)
- 확인 화면에 "**담당 설계사가 24시간 내 연락드립니다**" 메시지
- GA4 이벤트 전송: `lead_submitted` + `product_interest`

## 4. 개인정보처리방침 — 표준 목차

AIA·키움에셋·메타리치의 표준 구조를 기반으로 한 **11개 조** 템플릿. 클라이언트별 문구 교체.

1. **제1조 (개인정보의 처리 목적)** — 보험 상담, 마케팅(선택), 고객 관리
2. **제2조 (개인정보의 처리 및 보유 기간)** — 상담 목적 3년, 법령 별도 기간 명시
3. **제3조 (처리하는 개인정보 항목)** — 필수/선택 분리 명시
4. **제4조 (개인정보의 제3자 제공)** — 제휴 보험사 목록, 위탁 업체 목록
5. **제5조 (개인정보 처리 위탁)** — 호스팅, 결제, CS 등
6. **제6조 (정보주체의 권리·의무 및 행사 방법)** — 열람·정정·삭제·처리정지 요구권
7. **제7조 (개인정보의 파기)** — 방법, 절차, 기한
8. **제8조 (개인정보의 안전성 확보 조치)** — 암호화, 접근권한, 물리적 조치
9. **제9조 (쿠키 및 자동 수집 장치)** — GA4/Hotjar/픽셀 사용 명시, 거부 방법
10. **제10조 (개인정보 보호책임자)** — 성함·연락처·이메일
11. **제11조 (방침의 변경)** — 공고 방식, 시행일

### 4.1 GA 특화 체크포인트
- **제4조(제3자 제공)에 "제휴 보험사 N개사" 를 명시** 하고 별도 페이지(`/legal/partners`)에 전체 리스트.
- **제5조(위탁)에 "보험 상담 연결을 위한 설계사 조직"** 위탁 내용 반드시 기재.
- **제8조에 "신용정보 수집 시 별도 암호화"** 명시 (신용정보법 연동).
- **보존 기간을 "상담 완료 후 3년"** 으로 통일 (영업 목적).
- **마케팅 별도 동의 보유 기간**은 "동의 철회 시까지" 명시.

## 6. 보험업법 준수 체크포인트 (GA 특화)

| # | 조항 | 필수 조치 | 랜딩 구현 |
|---|---|---|---|
| 1 | §87 | 보험대리점 등록 확인 | Footer에 등록번호 표기 |
| 2 | §95 | 모집 관련 고지 | 리드폼 제출 시 "상담 연결 목적" 명시 |
| 3 | §95조의2 | 보험회사 명칭·상품명 정확 표기 | 협력사 로고월 + 상품 상세 |
| 4 | §95조의5 | 중요 사항 설명의무 | FAQ + 상품 상세 페이지 |
| 5 | §95조의6 | 설계사 자격 표시 (해당시) | 상담사 소개 섹션 (선택) |
| 6 | §97 | 모집 시 금지 행위 | **모집인 자격 없이 직접 체결 권유 금지** — 랜딩 카피에 "가입하세요" 식 직접 권유 문구 회피, "상담 신청" 으로 통일 |
| 7 | §102조의2 | 과장·허위 광고 금지 | 수치 표기(만족도%, 가입자수 등)는 근거일자·출처 병기 |

### 6.1 카피 작성 시 금지어·권장어

| 금지/주의 | 권장 대체 |
|---|---|
| "가입하세요" (직접 권유) | "무료 상담 신청", "내 보험 확인" |
| "최저가 보험", "가장 저렴한" | "합리적인", "비교 견적" |
| "100% 보장" | "제휴 보험사 중 맞춤 보장" |
| "보장 확정" | "상담을 통한 맞춤 설계" |
| "무조건", "반드시" | "일반적으로", "대부분의 경우" |

수치 표기는 **반드시 기준일자** 병기: "60만명이 선택한" → "60만명이 선택한 (2026년 3월 누적 기준)"
