# 체크리스트 01 — 요건정의 (Stage 0 Gate)

**용도**: `01_요건정의서_v0.1.md` 를 냉철하게 검토해서 Stage 0 Gate 통과 여부를 판단한다.
**사용법**: 각 항목을 독립적으로 체크하고, 하나라도 ❌ 면 그 항목만 수정 후 재검토한다. 감으로 OK하지 말고 **합격기준(Pass Criteria)** 을 근거로 판단한다.
**원칙**: 의심이 들면 ❌ 로 둔다. 넘어가도 되는 것과 넘어가면 안 되는 것을 분리한다.

## 부록: "이건 넘어가지 말아야 한다" — 시니어 관점 필살 체크 5개

GA·보험 랜딩을 실무에서 런칭할 때 **대체로 발목을 잡는** 5가지. 이 중 하나라도 모호하면 Stage 0을 끝내지 말 것.

1. **준법 고지 오류** — 보험업법 제2조의 "보험중개사·보험대리점" 정의에 따라 문구 잘못 쓰면 행정처분 소지. 클라이언트마다 확인 필수.
2. **개인정보 보관 기간 혼동** — "상담 목적 3년" vs "마케팅 활용 별도" 를 분리하지 않으면 과태료.
3. **리드 중복·허위 방지 전략 부재** — 경쟁사가 일부러 허위로 폼을 도배하는 사례 있음. rate limit + honeypot + 휴대폰 인증 중 최소 하나 필요.
4. **이탈률 높은 섹션 순서** — 히어로 바로 아래 리드폼이 맞는지, 아니면 "신뢰 요소(협력사·통계)" 가 먼저 오는 게 맞는지. 이 결정은 A/B 테스트로 검증한다.
5. **"왜 우리인가" 의 논리** — 보험업 경쟁 치열. "전문 설계사 N명 / 평균 상담 만족도 X / 제휴 보험사 Y개" 같은 **수치 기반 차별화** 가 없으면 감성 카피만으로는 전환율 떨어짐.
