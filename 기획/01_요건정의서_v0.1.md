## 0. 문서 목적

이 문서는 Stage 0(킥오프)의 산출물로서, 프로젝트의 **범위·방향·제약·성공기준**을 확정해 후속 Stage(리서치→IA→디자인→개발→QA)의 근거가 되도록 작성한다. 후속 문서(IA, 기능정의서, 디자인 시스템 매핑, 체크리스트)는 모두 이 문서를 기준으로 파생된다.

## 1. 프로젝트 개요

### 1.1 배경
- 발주처는 광고대행사(엠플래닛 추정)로, 보험·GA사 클라이언트를 대상으로 퍼포먼스 마케팅을 제공하고 있다.
- 단일 클라이언트 맞춤 랜딩이 아니라, **여러 클라이언트(AIA, 흥국, 메타리치, 캐롯, 삼성화재 다이렉트, DB손해보험, 메리츠화재, 키움에셋 등)에 반복 재활용 가능한 랜딩페이지 템플릿**을 제작하고, 나아가 이를 운영 가능한 **"랜딩플랫폼"** 으로 확장한다.
- 본 Stage는 그 플랫폼을 구성할 랜딩페이지의 **"표준 템플릿 v1"** 을 정의·제작하는 단계다.

### 1.2 목표(Goal)
1. 클라이언트별 **브랜드 테마(컬러·로고·타이틀·카피·상품 데이터·약관)만 교체**하면 배포 가능한 표준 템플릿을 확정한다.
2. 신뢰·전문형 톤으로, 보험·금융 업종에서 **리드 전환(상담 신청)** 을 최대화하는 UX 구조를 설계한다.
3. 보험업법·개인정보보호법·전자상거래법의 **필수 법적 고지**를 구조적으로 수용한다.
4. 후속 운영을 위해 **리드 DB + 관리자 페이지 + 게시판(사용자 문의 + 관리자 답변)** 의 최소 백엔드를 연동한다.

### 1.3 비목표(Non-Goal)
- 단일 클라이언트 전용 맞춤 마이크로사이트
- CMS 풀빌드(별도 프로젝트로 분리)
- 보험 상담·견적 자동화 계산기 (향후 확장)
- 실제 결제/청약 기능 (GA의 역할은 리드 전달까지)

## 5. 타깃 전략 (상품 주도 페르소나)

Jung님 지시대로 **타깃은 클라이언트가 아닌 상품군이 결정**하는 구조로 설계한다. 따라서 페르소나는 "상품 × 주요 니즈" 매트릭스로 정의한다.

| 상품군 | 대표 페르소나 | 핵심 니즈 | 주요 카피 방향 |
|--------|--------------|-----------|----------------|
| 암보험 | 40~50대 가장, 가족력 우려 | 진단비·수술비 보장 | 가족에 대한 책임 |
| 실손의료비 | 전 연령 | 병원비 실비 보장 | 일상 속 의료비 안전망 |
| 종신/정기 | 3040 가장 | 유족 생활비 보장 | 남겨질 가족의 삶 |
| 자동차·운전자 | 20~60대 운전자 | 사고 보장·벌금·형사 | 불시의 사고 대비 |
| 치아·어린이 | 학부모, 자녀 케어 | 치료비 부담 감소 | 아이의 미소, 치료비 부담 |
| 간병·치매 | 5060대 자녀, 6070 본인 | 장기 요양 대비 | 긴 간병의 무게를 덜다 |
| 자산형성(연금·저축) | 3050 직장인 | 은퇴 자산 | 미래를 설계하다 |

> 서브페이지 Hero·추천대상·카피는 이 매트릭스에서 꺼내 쓴다.

## 6. 테마 토큰 정책 (멀티 클라이언트 대응)

### 6.1 교체 대상 변수
```
--brand-primary        : 클라이언트 메인 컬러
--brand-primary-strong : hover/active
--brand-primary-soft   : 배경 틴트
--brand-secondary      : 보조 컬러
--brand-logo           : 로고 이미지 경로
--brand-name           : 브랜드명
--brand-font           : (선택) 브랜드 전용 폰트
```
### 6.2 교체 대상 콘텐츠 (JSON 스키마 초안)
```json

  "brand": {

    "logoUrl": "/assets/brands/aia/logo.svg",
    "primaryColor": "#D4002E",
    "secondaryColor": "#1A1A1A"
  },
  "hero": {
    "headline": "...",
    "subheadline": "...",
    "cta": { "label": "무료 상담 신청", "target": "#leadForm" }
  },
  "products": [
    { "slug": "cancer", "name": "암보험", "icon": "...", "description": "..." }
  ],
  "partners": [ { "name": "삼성생명", "logoUrl": "..." } ],
  "legal": {
    "companyName": "...",
    "registrationNumber": "...",
```

## 7. 리드폼 설계 원칙

### 7.1 필드 (권장 초안 — Jung님 합의 필요)
| 필드 | 유형 | 필수 | 비고 |
|------|------|------|------|
| 성함 | text | ✓ | 2자 이상 |
| 연락처 | tel | ✓ | 010-xxxx-xxxx 검증 |
| 생년월일 | date/select | ✓ | 연령 기반 상품 추천 근거 |
| 관심 상품 | select | ✓ | 상품군 중 택1 (서브에서는 고정) |
| 상담 희망 시간 | select | △ | 오전/오후/상관없음 |
| 기타 문의 | textarea | ✗ | 선택 |
| 개인정보 수집·이용 동의 | checkbox | ✓ | 필수 |
| 마케팅 활용 동의 | checkbox | ✗ | 선택 |

### 7.2 검증·UX 원칙
- 모바일 최적화(1컬럼), 순차 포커스, 자동 포맷팅
- 제출 성공 시 "담당 설계사가 24시간 내 연락드립니다" 메시지 + GA 이벤트 전송
- 중복 제출 방지 (같은 번호 24시간 1회)

### 7.3 관리자 측
- 리드 목록 조회·검색·필터(날짜·상품·상태)
- 엑셀 내보내기
- 상태 관리: 신규 / 상담중 / 완료 / 취소

## 8. 기술 스택 확정안

Jung님 선택("순수 HTML/CSS/JS + 간단 백엔드")에 **플랫폼화 대응 보정**을 더한 최종 안.

| 계층 | 선택 | 이유 |
|------|------|------|
| 프론트 마크업 | HTML + EJS 템플릿 | 섹션 양산·공통영역 1회 수정 가능 |
| 스타일 | CSS + CSS Variables | 토큰 기반 테마 스왑 |
| 스크립트 | Vanilla JS (+경량 라이브러리) | 의존성 최소화 |
| 빌드 | 단순 Node 스크립트 or 11ty | 선택 가능. MVP는 Node+EJS 추천 |
| 백엔드 | Node.js + Express | 학습·유지비용 낮음 |
| DB | SQLite (MVP) → MySQL (운영) | 초기 경량, 확장 가능 |
| 관리자 | 서버사이드 렌더링(EJS) | 별도 SPA 불필요 |
| 호스팅 | 도메인+VPS 또는 AWS Lightsail | 클라이언트별 서브도메인 운영 가능 |
| 분석 | GA4 + 리드폼 자체 이벤트 로그 | 전환 추적 |

> **Jung님 명시 확인 요청**: 위 "EJS + Node/Express + SQLite" 조합을 공식 기술스택으로 확정해도 되는지요?
