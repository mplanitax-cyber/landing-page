# 07. 통합 검토 체크리스트 — 랜딩 전체 Gate Review

**작성일**: 2026-04-24
**작성자**: 시니어 기획 (Senior Planner Agent)
**용도**: 요건정의 → IA → 디자인 → 개발 → QA → 런칭 **전 Stage 를 하나의 문서에서 냉철하게 검수**
**사용법**: 각 섹션별로 통과기준을 검토. **가중치 ★** 가 높은 항목은 미충족 시 Stage Gate 통과 불가. 하나라도 ❌ 면 해당 섹션만 재작업.
**규칙**: 의심되면 ❌, "대체로 되어있다" 는 ❌. 근거 자료를 인용하거나 직접 확인한 것만 ✅.

---

## 0. 사용 가이드

```
컬럼 정의:
- #         : 항목 번호
- 항목      : 검수 포인트
- 통과 기준 : 명확한 측정 기준
- 가중치    : ★ (감점) / ★★ (블로커) / ★★★ (런칭 불가)
- 담당자    : P(기획) / D(디자인) / F(프론트) / B(백엔드) / Q(QA) / L(법무)
- 상태      : ⬜ 미확인 / ✅ 통과 / ❌ 미통과 / ⏸ 보류 (Jung 승인 대기)
- 근거      : 파일·라인·스크린샷·링크
```

**Gate 판정 규칙**: 각 섹션에서 ★★★ 가 1건이라도 ❌ 또는 ⬜ 이면 Gate 불통과.
