## 0. 구조 원칙

### 0.1 고정(Fixed) vs 가변(Configurable)
모든 컴포넌트는 아래 세 가지 중 하나로 분류된다.
- **FIXED**: 모든 클라이언트에 동일 (레이아웃·순서·애니메이션·폰트)
- **THEMED**: CSS 변수로 교체 (컬러·로고·폰트)
- **CONTENT**: JSON으로 교체 (카피·이미지·상품 리스트·협력사·법적 고지)

### 0.2 페이지 렌더링 흐름
```
templates/*.ejs  +  config/{client}.json  +  tokens/{client}.css
        │                    │                        │
        └────────── build ──────────┴─── deploy ──────┘
                    │
                public/{client}/*.html
```

### 2.2 섹션별 상세 스펙

#### §1. GNB
- **구조**: 로고 / 상품 드롭다운 / 고객센터 / 상담신청(CTA 버튼)
- **모바일**: 햄버거 → Drawer
- **스크롤 시**: 상단 고정 + 배경 opacity 증가
- **JSON 필드**:
  ```json
  "gnb": {
    "menu": [
      { "label": "상품", "submenu": [...] },
      { "label": "회사소개", "href": "/about" },
      { "label": "고객센터", "href": "/board" }
    ],
    "cta": { "label": "무료 상담", "target": "#leadForm" }
  }
  ```

#### §2. Hero
- **구조**: 좌측 카피(eyebrow + headline + sub + CTA) | 우측 리드폼 카드 (split 60:40)
- **모바일**: 카피 위, 리드폼 아래 (1컬럼)
- **배경**: FIXED 그라데이션 (브랜드 primary → 투명) + 선택 이미지
- **JSON 필드**:
  ```json
  "hero": {
    "eyebrow": "보험 통합 비교 서비스",
    "headline": "30개 보험사 상품을 한 번에 비교하세요",
    "subheadline": "3초 만에 내게 맞는 보험 찾기",
    "cta": { "label": "무료 상담 신청", "target": "#leadForm" },
    "image": "/assets/{client}/hero.png"
  }
  ```

#### §3. 리드폼 ① (Hero 인접)
- **필드**: 성함*, 연락처*, 생년월일*, 관심상품*
- **동의**: 필수 2개(개인정보 수집·제3자 제공) + 선택 1개(마케팅)
- **검증**: 연락처 010-0000-0000 정규식, 생년월일 YYYY-MM-DD
- **제출**: `POST /api/leads` → 성공 시 확인 모달 + GA4 이벤트
- **중복 방지**: 동일 번호 24시간 1회
- **기술 스펙**:
  - Honeypot 필드 (봇 차단)
  - reCAPTCHA v3 (선택)
  - rate limit: 동일 IP 분당 3회

#### §4. 왜 비교해야 하는가 / 왜 우리인가
- **구조**: 섹션 타이틀 + 4개 수치 카드 그리드 (2×2 또는 4×1)
- **카드 템플릿**: 아이콘 + 큰 숫자 + 설명 + 기준일자 (소형)
- **JSON 필드**:
  ```json
  "whyUs": {
    "title": "왜 저희와 함께해야 할까요?",
    "cards": [
      { "icon": "...", "value": "30+", "label": "제휴 보험사", "asOf": "2026.04 기준" },
      { "icon": "...", "value": "12,000+", "label": "누적 상담 건수", "asOf": "2026.03 기준" },
      { "icon": "...", "value": "96.8%", "label": "고객 만족도", "asOf": "2025년 자체 조사" },
      { "icon": "...", "value": "10년", "label": "운영 경력", "asOf": "설립 2016" }
    ]
  }
  ```

#### §5. 상품 안내
- **구조**: 상품 카드 그리드 (데스크톱 4컬럼, 모바일 2컬럼)
- **카드**: 아이콘 + 상품명 + 한 줄 설명 + "자세히 보기" 링크 → `/products/{slug}`
- **JSON 필드**:
  ```json
  "products": [
    { "slug": "cancer", "name": "암보험", "icon": "...", "description": "진단비·수술비 든든하게" },
    { "slug": "medical", "name": "실손의료비", "icon": "...", "description": "병원비 실비 보장" },
    ...
  ]
  ```

#### §6. 협력사 로고월
- **구조**: 섹션 타이틀 + 설명 + 로고 그리드 (데스크톱 6컬럼 × N행, 모바일 3컬럼)
- **분류**: 생명보험 / 손해보험 2탭 (선택적)
- **클릭 동작**: 없음(트러스트 목적) 또는 해당 보험사 공식 사이트 새창
- **JSON 필드**:
  ```json
  "partners": {
    "title": "30개 보험사와 함께합니다",
    "description": "검증된 제휴 보험사만 비교합니다",
    "categories": {
      "life": [ { "name": "삼성생명", "logo": "/assets/partners/samsung-life.svg" }, ... ],
      "nonlife": [ { "name": "DB손해보험", "logo": "/assets/partners/db.svg" }, ... ]
    }
  }
  ```

#### §7. 절차 프로세스 (5단계)
- **구조**: 가로 타임라인 (데스크톱 5스텝 가로, 모바일 세로 스크롤)
- **스텝**: 1.상담 신청 → 2.설계사 배정 → 3.맞춤 설계 → 4.가입 → 5.사후 관리
- **각 스텝**: 아이콘 + 번호 + 타이틀 + 한 줄 설명
- **JSON 필드**: 기본값 제공, 클라이언트가 커스텀 가능

#### §8. 상담사 소개 (선택 on/off)
- **구조**: 카드 슬라이더 (데스크톱 3~4명 가로, 모바일 1명씩 가로 스크롤)
- **카드**: 상담사 사진 + 이름 + 직급·경력 + 전문 분야 + 한 줄 멘트
- **JSON 필드**:
  ```json
  "consultants": {
    "enabled": true,
    "items": [ { "name": "김보험", "title": "수석설계사", "years": "15년", "photo": "...", "specialty": "암·종신", "quote": "..." } ]
  }
  ```

#### §9. 데이터 교육 섹션 (선택 on/off)
- **구조**: 다크 배경 + 차트(막대 or 라인) + 설명 카피
- **차트**: Chart.js 또는 정적 SVG
- **JSON 필드**:
  ```json
  "education": {
    "enabled": true,
    "title": "왜 지금 이 보험이 필요한가?",
    "chart": { "type": "bar", "labels": [...], "values": [...] },
    "body": "연간 암 진단 비용이 1,450만원에 이르며..."
  }
  ```

#### §10. 리드폼 ② (본문 하단)
- **배경**: 다크 네이비 (신뢰·전문형 톤)
- **필드**: 성함*, 성별, 생년월일*, 연락처*, 관심상품*, 상담 희망시간, 기타 문의
- **동의**: 필수 2 + 선택 1
- **CTA**: "맞춤 보험 설계 받기"

#### §11. FAQ TOP5
- **DYNAMIC**: 서버에서 게시판 DB 조회 → 조회수/중복문의 TOP5 집계
- **아코디언**: 클릭 시 펼침
- **API**: `GET /api/faq/top5?client={slug}&product=all`
- **캐싱**: 1시간 캐시 (Redis 또는 인메모리)
- **Fallback**: API 실패 시 JSON 기본 FAQ 노출

#### §12. Footer
- 섹션 상단: 이용약관·개인정보·마케팅수신동의 링크 볼드
- 중단: 사업자 정보 블록 + 준법감시인확인필 + 주소·연락처
- 하단: 저작권 + 호스팅 제공자

## 4. 공통 페이지 IA

### 4.1 이용약관 (`/legal/terms`)
- 좌측 목차(데스크톱) + 우측 본문 (모바일은 상단 목차 → 본문)
- 각 조 앵커 링크 가능
- 이전 버전 열람 (`?version=2024-01`) 링크 하단

### 4.2 개인정보처리방침 (`/legal/privacy`)
- 상동
- 개인정보 보호책임자 카드 (상단 또는 사이드)
- 동의 철회 요청 버튼 → 폼 모달

### 4.3 게시판 (`/board`)
- **목록**: 제목·작성일·답변 상태(대기/답변완료)·조회수
  - 답변 상태 필터, 키워드 검색
- **상세**: 작성자(본인만 노출), 내용, 답변 (관리자)
- **작성**: 제목, 내용(에디터), 비밀글 옵션, 첨부(이미지 3장 제한)
- **인증**: 작성은 휴대폰 인증 또는 간단 캡챠
- **권한**:
  - 사용자: 본인 글만 수정·삭제 (작성 시 받은 토큰)
  - 관리자: 전체 CRUD + 답변 작성

## 5. 관리자 페이지

### 5.1 대시보드 (`/admin/`)
- 최근 7일 리드 카운트, 상품별 리드 비중, 게시판 대기 답변 수, FAQ TOP5 변화

### 5.2 리드 관리 (`/admin/leads`)
- 테이블: 일시 · 성함 · 연락처 · 관심상품 · 상태 · 비고
- 필터: 기간, 상품, 상태
- 액션: 상세 · 상태 변경(신규/상담중/완료/취소) · 엑셀 내보내기
- 검색: 이름·연락처

### 5.3 게시판 관리 (`/admin/board`)
- 목록: 답변 대기 우선 정렬
- 답변 작성 에디터
- 카테고리 관리(태그)

### 5.4 FAQ TOP5 집계 (`/admin/faq`)
- 집계 기간 설정 (기본 30일)
- 상위 질문 자동 추출 → 메인 FAQ 섹션에 반영
- 수동 오버라이드 가능

### 5.5 클라이언트 설정 (`/admin/config`)
- JSON 편집 GUI 또는 파일 업로드
- 미리보기 + 발행
- 배포 이력 (롤백 가능)

```
             content, is_private, view_count, created_at)
board_comments (id, post_id, author_type, content, created_at)  -- author_type: admin
faq_cache (client_id, product_slug, data_json, updated_at)
admin_users (id, email, password_hash, role, created_at)
admin_logs (id, user_id, action, target, meta_json, created_at)
```

### 6.2 인덱스
- `leads(client_id, created_at DESC)` — 리드 목록 정렬
- `leads(phone)` — 중복 방지
- `board_posts(client_id, created_at DESC)` — 게시판
- `board_posts(content)` — 전문검색 (SQLite FTS5 또는 MySQL FULLTEXT)

### 6.3 민감정보 처리
- `phone`, `birth` — AES-256 암호화 저장
- `author_phone_hash` — SHA-256 해시만 저장 (동일인 확인용)
- 평문 검색 불가, 암호화 키는 환경변수 분리

## 9. 반응형·접근성·성능

### 9.1 브레이크포인트
- Mobile: ~ 480px
- Tablet: 481 ~ 768px
- Desktop: 769 ~ 1200px
- Wide: 1201px 이상

### 9.2 접근성 (WCAG 2.1 AA)
- 컬러 대비 4.5:1 이상 (본문), 3:1 이상 (UI)
- 키보드 네비게이션 전 구간
- 폼 레이블·오류 메시지 `aria-*`
- 이미지 `alt`, 아이콘 `aria-hidden`
- 모달 focus trap

### 9.3 성능 목표
- LCP < 2.5s
- CLS < 0.1
- INP < 200ms
- 이미지 WebP + lazy loading
- 폰트 preconnect, subset
- CSS·JS minify, gzip
