## 0. 이 업무가 필요한 이유 (Why)

현재 `design_v1.0/` 은 **고정된 1개 랜딩**이다. 클라이언트가 10개로 늘면 40개 HTML 을 수동 관리하는 재앙이 온다. 이 프로젝트의 전략은 **섹션 컴포넌트 라이브러리 + JSON 데이터 교체** 구조. 즉:
- 1번 만든 템플릿을 10개 클라이언트에 재활용
- 브랜드 컬러·로고·상품·법적 고지는 `{client}.json` 한 파일로 제어
- 섹션 구성은 `sections: ["hero_right_sticky", "trust_4card", "partners_2col", ...]` 배열로 골라 담기

본 업무는 이 구조를 **v1 정적 랜딩을 해체·재조립하여 Nunjucks 템플릿 + Node 빌드 스크립트**로 만드는 일이다. 디자이너가 4/28 까지 레퍼런스 60개를 Figma 보드로 제출하고, 그 중 variant 1~2개씩을 우리 라이브러리에 탑재하게 된다.

## 5. 코드 컨벤션

### 5.1 HTML / Nunjucks
- `<html lang="ko">` 필수
- Semantic tag 우선: `<header>`, `<main>`, `<section>`, `<article>`, `<nav>`, `<footer>`
- Skip link `<a class="skip-link" href="#main">본문 바로가기</a>` 항상 첫 body 요소
- ARIA: 폼에 `<label for>`, 에러 `role="alert"`, 모달 `role="dialog" aria-modal`
- Heading 위계: H1은 페이지당 1개, H2는 섹션 타이틀, H3은 카드 타이틀
- Nunjucks partial 한 파일은 **1 섹션** 만 담당. 2 섹션 한꺼번에 금지

### 5.2 CSS
- **CSS Variables 필수**. 하드코딩 컬러 1줄이라도 있으면 PR 리젝트
- `.block__element--modifier` BEM-like 네이밍
- Mobile-first: 기본은 모바일, 미디어 쿼리는 `@media (min-width: 480px|768px|1200px)` 만 사용
- Logical properties 선호: `margin-inline-start` > `margin-left`
- `!important` 금지 (변수 재정의로 해결)
- 한 CSS 파일은 **1 컴포넌트** 만 담당 (BEM block 1개)
- `z-index` 는 tokens.css 에 정의된 스케일만 사용 (`var(--z-modal)` 등)

### 5.3 JavaScript
- ES2022 module (`import`/`export`)
- `querySelector` 는 한 번만 호출, 결과는 변수 캐시
- 이벤트 리스너는 **항상 cleanup 가능한 형태**로 등록
- `any`·globals·jQuery 금지
- 콜백 3단 이상 중첩 금지 (async/await 사용)
- fetch 는 try/catch + 사용자 친화 에러 메시지
- Formspree 응답 실패 시 **폼 데이터 보존** + 재시도 제안
- GA4 이벤트: `analytics.js` 에서 통합 관리. inline `gtag(...)` 호출 금지

### 5.4 Nunjucks 컨벤션
```njk
{# partial: leadform_right_sticky.njk
   props:
     - leadForm.title (string, optional)
     - leadForm.fields (array)
     - leadForm.consent (object)
     - formspree.formId (string, required)
#}
<aside class="leadform leadform--right-sticky" id="leadForm">
  <h2 class="leadform__title">{{ leadForm.title or "1분 상담 접수" }}</h2>
  <form class="leadform__form"
        method="POST"
        action="https://formspree.io/f/{{ formspree.formId }}">
    {% for field in leadForm.fields %}
      {% include "partials/_field.njk" %}
    {% endfor %}
  </form>
</aside>
```

- Partial 파일 상단에 **JSDoc-like 주석** 으로 props 명시
- `{# #}` 주석 활용, HTML 주석은 최소화
- 필수 props 가 없으면 빌드 실패시키기 (lint rule)

## 7. Do / Don't (Hard Rules)

### Do
- Semantic HTML5 + ARIA 패턴
- CSS Variables + BEM + Mobile-first
- ES2022 모듈·async/await
- **한 partial = 한 섹션**
- 에러 케이스 방어 (fetch 실패·폼 에러·네트워크 타임아웃)
- Conventional Commits (`feat(leadform): add as-hero variant`)
- 모든 PR 에 스크린샷 + Lighthouse 점수

### Don't
- **React / Vue / Svelte / Angular / Next.js 도입 금지**
- **Tailwind / Bootstrap / Material UI 금지** (CSS Variables 원칙 훼손)
- **jQuery / Lodash (전체) / Moment.js 금지** (번들 크기)
- **!important 금지** (CSS Variables 재정의로 해결)
- **div-soup 금지** (semantic tag 무시 금지)
- **`console.log` 프로덕션 잔재 금지**
