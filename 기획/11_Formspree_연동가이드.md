## 1. Formspree 가입 (3분)

### 1.1 회원가입
1. https://formspree.io 접속
2. 우측 상단 **Sign up** 클릭
3. 이메일 + 비밀번호 입력 (또는 Google 계정 연동)
4. 이메일 인증 링크 클릭 완료

> 무료 플랜(Free) 기준: **월 50건 리드 수신**, 기본 스팸 필터, 자동 이메일 알림. v1 MVP 에는 충분.
> 유료 플랜(Gold $10/월): 월 1,000건 + Custom domain + 파일 업로드. 런칭 후 트래픽 보고 전환.

### 1.2 폼 생성
1. 대시보드에서 **+ New Form** 클릭
2. Form 이름: `엠플랜잇 GA 랜딩 리드폼 v1` (자유)
3. Send To (수신 이메일): **리드 받을 이메일 주소** (운영 담당자·CRM 자동전달 주소)
4. Create Form 클릭

### 1.3 Form ID 확보
- 생성된 폼 설정 화면 상단 **Your form's endpoint** 항목에:
  ```
  https://formspree.io/f/xxxxxxxx
  ```
  형태의 URL 표기됨. 마지막 8자리(`xxxxxxxx`) 가 **Form ID**.
- 이 ID 를 복사하여 PM 에게 전달.

> **주의**: ID 는 공개돼도 안전하나, **수신 이메일 주소를 HTML 에 직접 노출하지 않도록** Formspree 를 쓰는 것임. ID 는 안전하게 공유 가능.

## 3. 테스트 리드 5건 체크리스트 (5분)

치환 완료 후, 브라우저에서 `design_v1.0/index.html` 열어 직접 테스트.

| # | 테스트 | 기대 동작 | 확인 |
|---|---|---|---|
| 1 | **정상 제출** — 모든 필드 정상 입력 | "상담 신청 완료" alert + Formspree 대시보드에 1건 수신 + 수신 이메일 1통 | ⬜ |
| 2 | **연락처 형식 오류** — "010-abc-1234" | 제출 차단 + 오류 alert | ⬜ |
| 3 | **필수 동의 미체크** — 개인정보 체크 해제 | 제출 차단 | ⬜ |
| 4 | **2차 리드폼(하단 다크)** — 별도 제출 | 상동. 첫 제출과 구분되도록 `form_position` 필드값 "secondary" 확인 | ⬜ |
| 5 | **서브페이지(암보험)** 제출 | `product` 필드가 자동 "암보험" 으로 수신되는지 | ⬜ |

### 3.1 Formspree 대시보드에서 확인
- 좌측 메뉴 **Submissions** → 방금 제출한 건 리스트 노출
- 각 필드(성함·연락처·관심상품 등) 값 정확 수신 여부
- `referrer` 헤더로 어느 페이지에서 왔는지 확인 가능

### 3.2 이메일 알림 확인
- Formspree 설정한 수신 이메일에 즉시 도착
- 스팸함도 확인 (첫 1~2건은 간혹 스팸 분류)

## 4. 알려진 한계·주의

1. **무료 플랜 월 50건 제한** — 초과 시 제출 차단. 런칭 전 플랜 업그레이드 또는 자체 백엔드(v2) 이관
2. **스팸 봇 공격** — 무료 플랜 기본 reCAPTCHA 있음. 추가 Honeypot 필드는 이미 스크립트에 있음
3. **수신 이메일 변경** — Formspree 대시보드 → 해당 폼 → Settings 에서 교체 가능 (HTML 재배포 불필요)
4. **GA4 이벤트 전송** — Formspree 는 GA4 통합 기능 없음. `script.js` 에서 `gtag('event', 'lead_submitted', {...})` 로 별도 처리 (현 script.js에 이미 반영)
5. **중복 방지** — Formspree 자체는 중복 필터 없음. 자체 백엔드 v2 때 처리. 현 단계에서는 동일 번호가 여러 번 도착해도 운영자가 수동 제거

## 5. 대안 (만약 Formspree 가 부적합 시)

| 대안 | 장점 | 단점 | 전환 비용 |
|---|---|---|---|
| **Netlify Forms** | Netlify 호스팅 시 무료 100건/월, 통합 편함 | Netlify 호스팅 전제 | 무료 (호스팅 변경 시 자동) |
| **Google Apps Script Webhook** | 완전 무료·무제한 | 개발자 설정 필요 | 1~2시간 |
| **EmailJS** | 프론트에서 직접 이메일 전송 | API 키 노출 리스크 | 30분 |
| **자체 백엔드 (v2 본격)** | 완전 제어·DB 저장 | 개발·운영 비용 | 2~3일 |

**현 단계 권장**: **Formspree 유지**. v1 MVP 에는 과도 엔지니어링 회피 원칙.
