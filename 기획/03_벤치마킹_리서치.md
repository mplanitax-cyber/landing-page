## 0. 리서치 요약 (Executive Summary)

본 프로젝트의 랜딩플랫폼은 **GA(General Agency) 포지션**이므로, 6개 벤치마크는 **역할에 따라 2군으로 분리** 해서 해석해야 한다.

- **A군 — GA/비교·중개 (제1 벤치마크)**: 키움에셋플래너, 메타리치
- **B군 — 직영 캐리어 (패턴 참고용)**: 캐롯, 한화손보 캐롯, 삼성화재 다이렉트, DB손해보험

A군이 "우리가 만들 것"에 해당하고, B군은 "리드 전환 UI 패턴" 을 학습하는 대상이다. 이 구분 없이 섞으면 GA 특유의 **제휴사·설계사·비교·리포트 발송** 요소를 놓치게 된다.

### 핵심 시사점 4가지
1. **GA 랜딩은 리드폼을 2회 이상 배치**한다 (Hero 인접 + 본문 중간 + 하단 긴 폼). 직영은 1회(sticky).
2. **GA는 "제휴사 로고월" 이 신뢰의 핵심 축**. 20~30개 노출이 표준.
3. **GA는 상담사/어드바이저 사진과 실명**을 전면에 배치 (메타리치). 직영은 고객 후기.
4. **카카오톡 연동(리포트·상담)** 이 GA의 차별점. 자사 DB에 저장하면서 카톡 봇으로 내용 전달.

## 2. 브랜드별 심층 관찰

### 2.1 캐롯 손보 (carrotins.com) — 직영 자동차

- **리드폼**: Hero 오른쪽 **sticky 사이드레일(440×624px)**. 헤드라인 **"1분 안에 보험료 확인"**. 필드: 이름, 주민번호, 휴대폰, 자동차번호. CTA **"보험료 확인"**. 법인 차량용 링크 보조.
- **히어로**: Lottie 다중 애니메이션 + 제품 이미지(캐롯 플러그 OBD).
- **결제/차별화**: 탭 2개(한 번에 결제 / 매월 후불 결제) + **월별 주행거리(210/48/64km) 시각화**.
- **할인 스택**: 7+ 특약(마일리지 5%, 자녀 32.3%, 무사고 14.5%, TMAP 20%, 커넥티드카, 차선이탈, 블랙박스…). "**자사 오프라인 대비 평균 22.3% 저렴**" 을 정면 카피로.
- **신뢰 요소**: **"10명 중 9명이 캐롯을 왜 다시 선택할까요?"** + 3개 후기. SK스피드메이트/히어로손해사정 파트너. **긴급출동 만족도 95.1%**.
- **컬러**: 캐롯 오렌지 + 옐로 액센트.
- **GA 관점 시사점**: 리드폼이 즉시 눈에 들어오는 배치가 결정적. 할인 스택은 수치 표기 방식을 그대로 차용 가능.

### 2.2 한화손보 캐롯 — 직영 (제휴 브랜드)

- 전체 구조는 carrotins.com 복제에 가까우나 **한화손해보험 준법 Footer 와 시티뷰 감성 섹션** 이 추가.
- 긴급출동 만족도 수치가 **98.1%** (본체는 95.1%)로 다름 → **캡처 시점별로 수치가 자주 업데이트되는 영역** 임을 시사. 랜딩에서 숫자는 운영 가능한 JSON 필드로 분리해야 함.
- **시사점**: 동일 UI를 다른 사업자 브랜드로 테마 스왑하는 사례. 우리 플랫폼의 모델 검증 사례.

### 2.3 삼성화재 다이렉트 — 직영 대형

- **히어로 카피**: "**대한민국 60만명이 선택한 / 삼성화재 운전자보험**" (25.11 누적). → **수치 기반 신뢰 공식**.
- **수치 교차판매**: "삼성화재 자동차보험 고객 / 매월 5% 할인".
- **상품 비교 섹션**: "**자동차보험과 어떤 점이 다른가요?**" 라는 split — 운전자 본인의 민·형사 책임 vs 차량/타인 손해 비교. → 보험 상품군 간 혼동 해소에 효과적.
- **가치제안**: Point 1/2 구조. 운전자벌금, 교통사고처리 지원금, 변호사 선임비용 **3대 비용**.
- **후기**: 6개 실명(마스킹) + 날짜(2026.04.03) + "**후기 전체보기**" CTA.
- **바텀 sticky CTA**: 하단 고정 바 **"보험료 계산"** 한 개. 하단 도달 시 즉시 전환 유도.
- **FAQ**: 4개, 아코디언.
- **준법**: 제26-1-1147호.
- **컬러**: 삼성 블루 + 뉴트럴 그레이. Conservative/corporate.
- **GA 관점 시사점**: **"숫자 + 날짜 + 후기 전체보기"** 조합은 GA에서도 차용 가능. "X개 보험사 / Y건 상담 / Z년 운영" 식.

### 2.4 DB손해보험 — 직영 대형 (암보험)

- **폼-as-Hero**: 스크롤 없이 hero 영역 자체가 리드폼 split — 왼쪽 일러스트, 오른쪽 상담 필드(생년월일, 성별, 운행차량, 직업). CTA **"보험료 계산"** + **"최근 계산 불러오기"**.
- **Dialog 모달**: 상품 상세를 풀스크린 Dialog 로 노출. 탭 5개(상품안내/가입안내/보장내용/가입예시/중요확인사항).
- **교차판매**: "자동차보험 고객님 보험료 매회 **1% 할인**".
- **준법**: 제2026-10095호.
- **GA 관점 시사점**: 상품 상세는 **페이지 이동 없이 모달** 로 처리하면 전환 흐름이 끊기지 않음. 우리 상품상세 FAQ 구간에서 적용 가능.

### 2.6 메타리치 — **GA 제2 벤치마크 (중개·상담 포지션)**

- **사업자 포지셔닝**: 설계사/어드바이저형 GA. "당신만의 / 보험 설계자, / 메타리치".

- **"왜 메타리치인가"**: 상담사 이름/포트레이트 + 신뢰 요소 bullet + 풀 쿼트.
- **제휴사 로고월**: "**제휴사 — 30개 보험사와 업무제휴**" (생명 + 손해 2컬럼). 하나생명, 신한라이프, 처브, 교보, 미래에셋생명, IBK연금, ABL, 한화생명, 삼성생명, KB손보, 현대해상, 농협생명, MetLife, 흥국, 한화손보, 알리안츠, DB, 삼성화재, Carrot, MG, AIG 등.
- **Footer**: 표준 GA 준법/사업자 정보.
- **컬러**: 스카이 블루 히어로 + 골드/앰버 메타리치 워드마크 + 다크 네이비 데이터·상담사 섹션.

## 3. GA 랜딩 표준 패턴 (A군 기준 재구성)

본 프로젝트 랜딩페이지의 **표준 패턴**은 아래 GA 공통 구조를 기반으로 한다.

### 3.1 메인 페이지 — GA 표준 순서 (확정 반영)

1. **GNB** — 상품 카테고리 8개 내외 + 회사소개/고객센터/상담신청 링크
2. **Hero (메인 카피 + 리드폼 ①)** — split 또는 sticky. CTA는 "무료 상담 신청" / "내 보험 확인"
3. **왜 비교해야 하는가 / 왜 우리인가** — 수치 기반 (N개 보험사, M건 상담, Y년 운영, 만족도 X%)
4. **상품 안내** — 카테고리 카드 그리드 (8개 상품군)
5. **협력사 로고월** — **30개 내외** (생명·손해 2컬럼 분리 권장)
6. **절차 프로세스** — 1단계 상담 신청 → 2단계 설계사 배정 → 3단계 맞춤 설계 → 4단계 가입 → 5단계 사후 관리
7. **상담사 소개 (선택)** — 클라이언트별 on/off
8. **데이터 교육 섹션 (선택)** — 통계·차트로 보험 필요성 설득
9. **리드폼 ②** — 다크 배경, 긴 폼. "더 자세한 상담을 원하신다면"
10. **FAQ TOP5** — 게시판 DB에서 자동 집계
11. **Footer** — 준법감시인확인필 + 사업자 정보 + 링크

### 3.2 서브 페이지 — 상품별 표준 순서

## 4. 우리 랜딩플랫폼이 반드시 차용/차별화할 것

- 모바일 최적화 (삼성식 sticky 바텀 CTA)

### 4.2 선택 차용 (Nice-to-have, 클라이언트 옵션)
- 상담사 사진·이름 (클라이언트에 실제 상담사 풀 있을 때만 on)
- 데이터 교육 섹션 (통계 자료가 있을 때만 on)
- 보험료 마스킹 비교카드 (법무 승인 후 on)
- 카카오톡 리포트/상담 연동 (v2)
- 상품 상세 Dialog 모달 (v2)

### 4.3 의도적으로 차별화할 것
- **GA 대행사 관점의 "한 번 만들면 여러 클라이언트에 재사용"** — 테마 토큰·JSON 스키마로 분리

- **실명 상담사 < 성과 수치** (상담사 풀이 클라이언트별로 다르므로 수치 중심으로 통일)
