## 섹션 1. GNB / 네비게이션

**등급**: ★★★★★ (Pattern A)
**태그 enum**: `sticky={y,n} / cta_count={0,1,2} / logo_position={left,center} / menu_depth={1,2}`

🟢 **채택 권고**

   **Pattern A — GA 심플 top-nav** (키움·메타리치·굿리치식)
   - 로고(좌) + 상품 카테고리 가로 탭(중앙) + 상담/채팅 CTA(우)
   - Sticky=Y (스크롤 시 상단 고정), 투명→솔리드 전환
   - tags: sticky=y, cta_count=1, logo_position=left, menu_depth=1

🔴 **제외**

   - **Pattern B 직영 2단 nav** (삼성화재·메리츠식): 상단 로그인바 + 메인 메뉴 + 상품 카테고리 아이콘 바 — GA엔 과잉
   - **Pattern C 핀테크 미니멀** (카카오페이·토스): 앱 설치 CTA — GA 상담 플로우와 맞지 않음

🟡 **주의**

   - 상품 탭 수는 **최대 6개**로 제한 (암·건강·실손·치매·청년·어린이). 9개 이상이면 스크롤 발생 → 모바일 UX 훼손
   - 드롭다운 depth 2단 이상 지양

📝 **이유**: GA는 상품 카테고리 직접 접근 가능성이 상담 전환의 1차 관문. 심플 탭이 최적

## 섹션 2. Hero — 메인 카피 + 시각 요소

**등급**: ★★★★★ (Pattern A)
**태그 enum**: `layout={split,full,stacked} / visual={photo,illust,lottie,none} / cta_count={0,1,2} / headline_lines={1,2,3}`

🟢 **채택 권고**

   **Pattern A — split 좌리드폼 + 우일러스트** (키움·메리츠·메타리치)
   - 좌: 리드폼 박스(white/shadow) - 필드 4개 + 약관 + CTA
   - 우: 아이소메트릭/캐릭터 일러스트 + 퍼플·블루 그라데이션
   - 헤드라인: 상단 2줄 이내, 보조카피 1줄
   - tags: layout=split, visual=illust, cta_count=1, headline_lines=2

🔴 **제외**

   - **Pattern D 다크 B2B 배너** (토스인슈식): GA 리드 전환과 톤 불일치
   - 실사진 full-bleed 단독(굿리치): 브랜드 소개용은 OK이나 전환 페이지엔 약함

🟡 **주의**

   - 일러스트는 Lucide + 자체 SVG 조합 권장 (외부 의존 최소화)
   - 비디오 배경은 v1 제외 (로딩 비용·모바일 데이터 이슈)

📝 **이유**: Split은 "시각적 주의 + 행동 유도"를 동시에 수행. 우측 일러스트는 시선 유도, 좌측 폼은 즉시 전환

**태그 enum**: `columns={2,3,4} / icon_style={line,filled,illust,photo} / card_height={fixed,flexible}`

   - 질문형 예시: "암 진단 후 바로 보장받을 수 있을까요?" / "운전자 보험이 왜 필요할까요?"

📝 **이유**: 질문형 카피는 사용자 니즈를 직접 자극해 클릭 동기 부여. 일러스트는 위계 + 브랜드 온도

## 섹션 8. 절차 프로세스 (Step Flow)

**등급**: ★★★★ (Pattern A)
**태그 enum**: `steps={3,4,5,6} / orient={horizontal,vertical} / marker={circle,square,icon}`

🟢 **채택 권고**

   **Pattern A — 가로 3~4단 타임라인 + 원형 번호**
   - 3단계: 상담 접수 → 맞춤 설계 → 가입 완료
   - 각 단계: 원형 번호(브랜드 컬러) + 미니 아이콘 + 타이틀 + 1줄 설명
   - 단계 간 연결선 (점선 또는 화살표)
   - orient=horizontal, steps=3, marker=circle

🔴 **제외**

   - **5단계 이상**: 복잡도 증가 + 전환 의욕 저하
   - **세로 카드 배치 (vertical)**: 모바일 외 데스크톱엔 공간 낭비

🟡 **주의**

   - 각 단계 설명은 **15자 이내 간결하게** (3단계 × 긴 설명 = 오히려 복잡)
   - 마지막 단계 "가입 완료"는 성취감 아이콘(✓) 활용

📝 **이유**: 3단계 구조가 심리적 완료 예상을 용이하게 함. 가입까지 "멀지 않다"는 메시지

📝 **이유**: 사람 얼굴 1명은 전체 팀 사진보다 "나만의 설계사" 감정 연결 강함
